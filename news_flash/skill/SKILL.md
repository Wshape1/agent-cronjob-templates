# 新闻快闪

## 核心目标

获取**最近最重要的不超过5条**财经新闻，不是全部新闻，是精选。

## 格式铁律

1. **标题**: `# 📰 新闻快闪 | YYYY.M.D`（不加"午间""晨间""晚报"等时间段标签）
2. **最多5条**，按重要性排序
3. **每条精确时间戳**: 用 `🕐` 标注，精确到小时或分钟，不加"北京时间"（默认即北京时间）
4. **简洁优先**: 内容不超过3行，不堆砌背景，不重复标题

## 单条格式模板

```markdown
### emoji 加粗短标题（4-8字概括核心）
**🕐 M月DD日 HH:MM**
一两句话核心事实和关键数据。

> 💬 **简评：** 一句话分析事件本质或市场意义。
> 🎯 **对你的影响：** 一句话说明对用户投资/生活的具体影响。
```

## 完整输出模板

```markdown
# 📰 新闻快闪 | YYYY.M.D

### emoji 短标题
**🕐 时间**
核心事实。

> 💬 **简评：** 本质分析。
> 🎯 **对你的影响：** 具体影响。

---

### emoji 下一条...
（共5条）

---

> ⚠️ 信息来源：财联社电报（cls.cn/telegraph）等，仅供参考，不构成投资建议。
```

## 选股标准

**以国内基金视角为核心**，优先选对A股基金有影响的事件：
- 🥇 第一梯队: A股收盘数据、央行政策、地缘冲突（影响A股的部分）
- 🥈 第二梯队: 美股/大宗商品对A股的传导、行业重大动态
- 🥉 第三梯队: 海外市场概况（仅保留对国内有传导意义的）

**跳过**: 纯海外政治新闻（除非影响A股）、纯公司公告（除非影响巨大）、广告、小市值消息

## 基金影响标注

每条新闻的"对你的影响"侧重**基金操作建议**：
- 哪些基金板块受利好/利空（半导体ETF、新能源ETF、黄金ETF、军工ETF等）
- 是否需要调仓、加减仓
- 短线 vs 中线判断

## 抓取方法

**用 web_fetch 并行抓取多个信源**：

1. web_fetch 抓取 `https://www.cls.cn/telegraph` → 解析嵌入文本
2. web_fetch 抓取 `https://finance.sina.com.cn/7x24/` → 解析滚动新闻
3. 合并去重，按重要性排序，取前5条

**web_fetch 不可用时 fallback**: 用 execute_code + requests 同样抓取

## 已知坑

- 财联社API需要签名 → 用web_fetch抓HTML页面后解析嵌入文本，或用requests抓HTML
- 新浪7x24是SPA → 优先用财联社，新浪作补充
- 36氪API不稳定 → 仅作第三备选
- 浏览器工具缺依赖 → 不要用 browser_navigate
- **Cron投递不要用 weixin** — iLink API 频率限制 (errcode -2) 导致消息发不出去，用 feishu 或 telegram
- **数据验证铁律** — web_fetch 返回空内容或只有导航/页头时，不要把搜索结果页的标题/摘要当成已验证数据来呈现。必须拿到实际文章正文才能引用具体数字。如果某个源抓不到正文，换下一个源，而不是用未验证的片段拼凑结论。用户明确要求"数据要基于网上权威信息，而不是自己捏造"
- **Cron脚本BOM清洗** — 财经网站抓取的数据经常包含不可见的 U+FEFF (BOM) 字符，会触发 cron 注入扫描器导致整个任务被 BLOCKED。采集脚本输出 JSON 前必须递归清除所有 `\ufeff`。症状：cron 状态=error，输出文件仅 ~500 字节，内容显示 "Blocked: prompt contains invisible unicode U+FEFF"。修复见 `web-scraping-aggregator` skill 的 Script Output Contract。
